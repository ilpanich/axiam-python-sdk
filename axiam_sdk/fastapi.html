axiam_sdk.fastapi

FastAPI Depends(...) dependency-injection integration (D-09, CONTRACT.md §10/§11).

Verifies an AXIAM access token LOCALLY (no per-request round-trip to the AXIAM server on a JWKS cache hit) via axiam_sdk._jwks.JwksVerifier and returns the authenticated identity (user_id, tenant_id, roles) for injection into a FastAPI route via Depends(...).

This module is a dependency-only integration (D-09) — there is deliberately no ASGI-middleware variant. It is imported ONLY as axiam_sdk.fastapi (never from the top-level axiam_sdk/__init__.py, see Anti-Pattern/T-19-22) so that pure-REST/gRPC/AMQP consumers of axiam-sdk are never forced to install fastapi.

Security-critical invariant — cross-tenant token replay defense (T-19-19): the AXIAM JWKS is organization-wide, not tenant-scoped, so a token that is signature-valid may still belong to a different tenant. This dependency enforces claims["tenant_id"] == configured_tenant BEFORE any claim is trusted further — mirrors the Go SDK's middleware/nethttp.go lines 78-95.

Security-critical invariant — expiry (T-19-20): JwksVerifier.verify() checks the signature only, not exp (documented on that class). This dependency independently rejects an expired-but-signature-valid token.

Security-critical invariant — no token leakage (T-19-21): no raw token value is ever included in any HTTPException detail.

CSRF (cookie double-submit, CONTRACT.md §3): a request authenticated via the axiam_access COOKIE is not CSRF-immune the way a Authorization: Bearer header request is — a cross-site attacker cannot set arbitrary request headers, but a same-site cookie is attached to cross-site requests automatically by the browser. For any cookie-sourced credential on a state-changing method (anything other than GET/HEAD/OPTIONS), this dependency additionally requires the X-CSRF-Token request header to be present and equal (constant-time) to the axiam_csrf cookie value, rejecting with 403 on mismatch/absence. This mirrors, locally, the same double-submit check the AXIAM server performs on its own endpoints (§3) — see also the Java SDK's .../AxiamAuthenticationFilter.java for the same pattern applied to the Spring integration.

CONTRACT.md §11 — declarative authorization helpers: require_access() and require_role() are an additive per-endpoint authorization layer built strictly on top of the authentication pipeline above (shared via the internal _authenticate() helper both require_authenticated_user() and require_access() compose with — the §10 verification path, JWKS, tenant check, CSRF, is never re-implemented or bypassed). require_access resolves a target resource UUID from the request (§11.3 precedence: resource_id literal, then resource_param path parameter, then a resolver callback), then calls ~axiam_sdk.AsyncAxiamClient's check_access(...) with subject_id set to the authenticated caller's user_id (§11.2) — never this SDK client's own, typically service-account, identity. Error mapping (§11.2, §2): unauthenticated -> 401 authentication_failed; denied or AuthzError -> 403 authorization_denied; missing/unparseable resource id -> 400 invalid_request; AuthError/NetworkError while calling the authz endpoint (the SDK client could not obtain a decision) -> fail closed with 503 authz_unavailable, per §11.2.5's "never allow on transport failure" rule. No decision caching (§11.2.6) — every call is a fresh authz-endpoint round-trip. Deny/error paths never log or echo the token (§11.2.8). require_role is a local, no-round-trip check against the already- verified identity's roles (§11.2.9) — it is NOT a substitute for require_access's authoritative, resource-level server check.

112@dataclass(frozen=True)
113class _Credential:
114    """A verified-candidate token plus whether it was sourced from the
115    ``axiam_access`` cookie (as opposed to the ``Authorization`` header) —
116    needed to decide whether the CSRF double-submit check applies."""
117
118    value: str
119    from_cookie: bool

122def _extract_token(request: Request) -> _Credential:
123    """Extract the bearer token from ``Authorization: Bearer <token>``,
124    falling back to the ``axiam_access`` session cookie.
125
126    Ported 1:1 from ``the Go SDK's middleware/nethttp.go``'s ``extractToken``
127    (lines 109-125): Bearer header first, cookie fallback second, standardized
128    401 with no token value on failure — this exact ordering is a Shared
129    Pattern (19-PATTERNS.md) also used by the Django middleware.
130    """
131    header = request.headers.get("authorization")
132    if header:
133        scheme, _, credentials = header.strip().partition(" ")
134        if scheme.lower() == "bearer" and credentials.strip():
135            return _Credential(credentials.strip(), from_cookie=False)
136        raise HTTPException(
137            status_code=_AUTH_FAILED_STATUS, detail="missing authentication credentials"
138        )
139
140    cookie = request.cookies.get("axiam_access")
141    if cookie:
142        return _Credential(cookie, from_cookie=True)
143
144    raise HTTPException(
145        status_code=_AUTH_FAILED_STATUS, detail="missing authentication credentials"
146    )

149def _assert_csrf_valid(request: Request) -> None:
150    """Cookie double-submit check (CONTRACT.md §3): the ``X-CSRF-Token``
151    request header must be present and equal, constant-time, to the
152    ``axiam_csrf`` cookie value. Raises ``HTTPException`` 403 on
153    mismatch/absence.
154
155    Only called for cookie-sourced credentials on state-changing methods
156    (see ``_dependency``) — a Bearer-header request is CSRF-immune by
157    construction, since a cross-site attacker cannot set arbitrary request
158    headers.
159    """
160    header = request.headers.get(_CSRF_HEADER_NAME)
161    cookie = request.cookies.get(_CSRF_COOKIE_NAME)
162    if not header or not cookie or not hmac.compare_digest(header, cookie):
163        raise HTTPException(status_code=_CSRF_FAILED_STATUS, detail="CSRF validation failed")

166async def _authenticate(
167    request: Request, verifier: JwksVerifier, configured_tenant: str
168) -> AxiamUser:
169    """The shared extract/CSRF/verify/exp/tenant authentication pipeline
170    (CONTRACT.md §10) underlying both :func:`require_authenticated_user` and
171    :func:`require_access` (CONTRACT.md §11.2.1: the declarative
172    authorization helper MUST run strictly after, and never duplicate or
173    bypass, this same verification path).
174
175    1. Extracts the token (Authorization Bearer, else ``axiam_access`` cookie).
176    2. When the credential is cookie-sourced AND the request method is
177       state-changing (not GET/HEAD/OPTIONS), enforces the CSRF
178       double-submit check (CONTRACT.md §3) before verification proceeds —
179       raises ``HTTPException`` 403 on failure.
180    3. Verifies it locally via ``verifier.verify()`` (signature + ``sub`` only).
181    4. Independently checks ``exp`` — the verifier does not (T-19-20).
182    5. Enforces ``claims["tenant_id"] == configured_tenant`` BEFORE trusting
183       any claim further (cross-tenant replay defense, T-19-19).
184    6. Returns :class:`AxiamUser` on success; raises ``HTTPException`` 401 on
185       any authentication failure, never including the raw token value.
186    """
187    credential = _extract_token(request)
188
189    if credential.from_cookie and request.method.upper() not in _SAFE_METHODS:
190        _assert_csrf_valid(request)
191
192    token = credential.value
193
194    try:
195        claims = verifier.verify(token)
196        # SDK-11: coerce ``exp`` to float INSIDE the verify try/except so a
197        # signature-valid token carrying a non-numeric ``exp`` (e.g. a
198        # string) maps to the normal invalid-token -> 401 path rather than
199        # a ValueError/TypeError propagating as an unhandled 500. Preserves
200        # the "malformed token -> 401" invariant (CONTRACT.md §10).
201        exp = claims.get("exp")
202        exp_ts = float(exp) if exp is not None else None
203    except Exception as exc:
204        raise HTTPException(
205            status_code=_AUTH_FAILED_STATUS, detail="invalid or expired token"
206        ) from exc
207
208    if exp_ts is not None and time.time() >= exp_ts:
209        raise HTTPException(status_code=_AUTH_FAILED_STATUS, detail="invalid or expired token")
210
211    # Cross-tenant replay defense (T-19-19): the JWKS is organization-wide,
212    # not tenant-scoped, so a signature-valid token may belong to a
213    # different tenant. MUST be enforced before any claim is trusted
214    # further (mirrors nethttp.go lines 78-95).
215    tenant_id = claims.get("tenant_id")
216    if not tenant_id or tenant_id != configured_tenant:
217        raise HTTPException(
218            status_code=_AUTH_FAILED_STATUS,
219            detail="token tenant_id does not match the configured tenant",
220        )
221
222    # WR-02: ``dict.get("scope", "")`` returns ``None`` (not ``""``) when
223    # the claim is PRESENT with an explicit JSON ``null`` value, and
224    # ``list(None)`` raises TypeError — which would propagate AFTER the
225    # verify() try/except above and surface as an unhandled 500 for an
226    # otherwise signature-valid token. ``claims.get("scope") or ""`` maps
227    # both absent AND null scope to empty roles, matching how an absent
228    # scope is already handled (CONTRACT.md §10: malformed claims must
229    # still degrade to a standardized 401, never a 500).
230    roles_claim = claims.get("scope") or ""
231    roles = roles_claim.split() if isinstance(roles_claim, str) else list(roles_claim)
232
233    subject = claims.get("sub")
234    if not subject:
235        raise HTTPException(status_code=_AUTH_FAILED_STATUS, detail="invalid or expired token")
236
237    return AxiamUser(user_id=subject, tenant_id=tenant_id, roles=roles)

240def require_authenticated_user(
241    verifier: JwksVerifier, configured_tenant: str
242) -> Callable[[Request], Awaitable[AxiamUser]]:
243    """Factory returning a ``Depends(...)``-compatible async dependency
244    (mirrors the Go middleware's ``Middleware(verifier, configuredTenant,
245    opts...)`` factory pattern, D-09).
246
247    Usage::
248
249        @app.get("/me")
250        async def me(user: AxiamUser = Depends(require_authenticated_user(verifier, "acme"))):
251            ...
252
253    The returned dependency delegates to :func:`_authenticate` for the full
254    extract/CSRF/verify/exp/tenant pipeline — see that function's docstring
255    for the exact steps. Returns :class:`AxiamUser` on success; raises
256    ``HTTPException`` 401 on any authentication failure, never including the
257    raw token value.
258    """
259
260    async def _dependency(request: Request) -> AxiamUser:
261        """The actual ``Depends(...)``-injected coroutine — delegates
262        entirely to :func:`_authenticate` (see :func:`require_authenticated_user`'s
263        docstring for the pipeline it runs)."""
264        return await _authenticate(request, verifier, configured_tenant)
265
266    return _dependency

269def _resolve_resource_id(
270    request: Request,
271    *,
272    resource_id: str | None,
273    resource_param: str | None,
274    resolver: Callable[[Request], str] | None,
275) -> str:
276    """Resolve and validate the target resource UUID per CONTRACT.md
277    §11.2.3's precedence order: a literal ``resource_id``, else the
278    ``resource_param`` path parameter, else the ``resolver`` callback.
279
280    A missing or unparseable resource value is a programming error surfaced
281    as ``HTTPException`` 400 (§11.2.5) — never a silent allow, never a
282    nil/empty-UUID fallback.
283    """
284    raw: object
285    if resource_id is not None:
286        raw = resource_id
287    elif resource_param is not None:
288        raw = request.path_params.get(resource_param)
289        if raw is None:
290            raise HTTPException(
291                status_code=_INVALID_REQUEST_STATUS,
292                detail={
293                    "error": "invalid_request",
294                    "message": f"missing path parameter {resource_param!r}",
295                },
296            )
297    else:
298        assert resolver is not None  # enforced by require_access at construction time
299        try:
300            raw = resolver(request)
301        except Exception as exc:
302            raise HTTPException(
303                status_code=_INVALID_REQUEST_STATUS,
304                detail={"error": "invalid_request", "message": "resource resolver failed"},
305            ) from exc
306
307    try:
308        uuid.UUID(str(raw))
309    except (ValueError, AttributeError, TypeError) as exc:
310        raise HTTPException(
311            status_code=_INVALID_REQUEST_STATUS,
312            detail={"error": "invalid_request", "message": "resource id is not a valid UUID"},
313        ) from exc
314    return str(raw)

317def require_access(
318    verifier: JwksVerifier,
319    configured_tenant: str,
320    client: AsyncAxiamClient,
321    action: str,
322    *,
323    resource_param: str | None = None,
324    resource_id: str | None = None,
325    resolver: Callable[[Request], str] | None = None,
326    scope: str | None = None,
327) -> Callable[[Request], Awaitable[AxiamUser]]:
328    """Factory returning a ``Depends(...)``-compatible async dependency that
329    requires an AXIAM authorization check for ``action`` on a resource
330    resolved from the request (CONTRACT.md §11).
331
332    Composes with, and runs strictly after, the same authentication pipeline
333    :func:`require_authenticated_user` uses (:func:`_authenticate` — never a
334    separate/duplicated token-verification path, §11.2.1).
335
336    Usage::
337
338        @app.get("/docs/{doc_id}")
339        async def get_doc(
340            doc_id: str,
341            user: AxiamUser = Depends(
342                require_access(verifier, "acme", client, "read", resource_param="doc_id")
343            ),
344        ):
345            ...
346
347    Exactly one of ``resource_id`` (a literal UUID, for singleton resources),
348    ``resource_param`` (a path parameter name), or ``resolver`` (a
349    ``request -> str`` callback for anything else — body fields, headers,
350    composite lookups) must be supplied (§11.2.3); this is validated eagerly
351    at factory-construction time, not per-request.
352
353    The returned dependency:
354    1. Authenticates the request via :func:`_authenticate` (401 on failure).
355    2. Resolves and validates the resource UUID (400 on a missing/unparseable
356       value, §11.2.3).
357    3. Calls ``client.check_access(action, resource_id, scope=scope,
358       subject_id=<authenticated user_id>)`` — ``subject_id`` is always the
359       *request's* authenticated caller, never this client's own (often
360       service-account) identity (§11.2.2).
361    4. Denied (``allowed=False`` or an ``AuthzError`` from the server, e.g. the
362       client lacks ``authz:check_as``) -> ``HTTPException`` 403
363       ``authorization_denied``.
364    5. A transport failure (``AuthError``/``NetworkError`` — the SDK could not
365       obtain a decision at all) -> **fail closed** with ``HTTPException`` 503
366       ``authz_unavailable`` (§11.2.5) — never allow, never retry beyond the
367       client's own bounded single-flight refresh.
368    6. On success, returns the same :class:`AxiamUser` so the handler keeps
369       the caller's identity (roles etc.) without a second lookup.
370
371    No decision caching (§11.2.6): every call performs a fresh
372    ``check_access`` round-trip. Deny/error paths never log or echo the raw
373    token (§11.2.8).
374    """
375    resource_sources = [
376        source for source in (resource_id, resource_param, resolver) if source is not None
377    ]
378    if len(resource_sources) != 1:
379        raise ValueError(
380            "require_access requires exactly one of resource_id, resource_param, or resolver"
381        )
382
383    async def _dependency(request: Request) -> AxiamUser:
384        """The actual ``Depends(...)``-injected coroutine — authenticate,
385        resolve the resource, then perform the authorization check (see
386        :func:`require_access`'s docstring for the full pipeline)."""
387        user = await _authenticate(request, verifier, configured_tenant)
388        target_resource_id = _resolve_resource_id(
389            request,
390            resource_id=resource_id,
391            resource_param=resource_param,
392            resolver=resolver,
393        )
394
395        try:
396            result = await client.check_access(
397                action, target_resource_id, scope=scope, subject_id=user.user_id
398            )
399        except AuthzError as exc:
400            raise HTTPException(
401                status_code=_AUTHZ_DENIED_STATUS,
402                detail={
403                    "error": "authorization_denied",
404                    "message": f"caller lacks permission for action {action!r}",
405                },
406            ) from exc
407        except (AuthError, NetworkError) as exc:
408            # §11.2.5: fail closed — a transport failure while calling the
409            # authz endpoint is "couldn't decide", never a silent allow.
410            raise HTTPException(
411                status_code=_AUTHZ_UNAVAILABLE_STATUS,
412                detail={
413                    "error": "authz_unavailable",
414                    "message": "authorization service unavailable",
415                },
416            ) from exc
417
418        if not result.allowed:
419            raise HTTPException(
420                status_code=_AUTHZ_DENIED_STATUS,
421                detail={
422                    "error": "authorization_denied",
423                    "message": f"caller lacks permission for action {action!r}",
424                },
425            )
426        return user
427
428    return _dependency

431def require_role(
432    verifier: JwksVerifier, configured_tenant: str, *roles: str
433) -> Callable[[Request], Awaitable[AxiamUser]]:
434    """Factory returning a ``Depends(...)``-compatible async dependency that
435    requires the authenticated identity's verified ``roles`` to contain at
436    least one of ``roles`` (CONTRACT.md §11).
437
438    A **local** check only — reads the claims already verified by
439    :func:`_authenticate`; it never calls the AXIAM server. Cheaper but
440    coarser than :func:`require_access`, and documented (§11.2.9) as NOT a
441    substitute for the authoritative, resource-level check: role names are
442    tenant-defined and callers should still gate any resource-specific
443    action behind :func:`require_access`.
444
445    Usage::
446
447        @app.delete("/admin/reset")
448        async def reset(user: AxiamUser = Depends(require_role(verifier, "acme", "admin"))):
449            ...
450    """
451
452    async def _dependency(request: Request) -> AxiamUser:
453        """The actual ``Depends(...)``-injected coroutine — authenticate,
454        then check the verified identity's roles locally (see
455        :func:`require_role`'s docstring)."""
456        user = await _authenticate(request, verifier, configured_tenant)
457        if not any(role in user.roles for role in roles):
458            raise HTTPException(
459                status_code=_AUTHZ_DENIED_STATUS,
460                detail={
461                    "error": "authorization_denied",
462                    "message": "caller lacks a required role",
463                },
464            )
465        return user
466
467    return _dependency

 35class AsyncAxiamClient(_AxiamClientBase):
 36    """The AXIAM SDK's dedicated async REST entry point (CONTRACT.md §1-§10,
 37    SDK-Q08).
 38
 39    ``await client.login(...)`` returns a typed
 40    :class:`~axiam_sdk._models.LoginResult` with ``mfa_required`` (SC#1) — the
 41    same contract as the sync :class:`~axiam_sdk.AxiamClient`, on its own
 42    dedicated async object. Constructing both an ``AxiamClient`` and an
 43    ``AsyncAxiamClient`` against the same ``base_url``/``tenant_slug`` gives
 44    each its own independent ``_Session`` (cookie jar, CSRF state, refresh
 45    guard) — the two classes do NOT share session state with each other, only
 46    each shares consistently within its own sync or async call path.
 47    """
 48
 49    # ------------------------------------------------------------------
 50    # Lifecycle (D-19)

 56    def __init__(self, base_url: str, *, lifespan: int = _DEFAULT_LIFESPAN_SECONDS) -> None:
 57        """Build a verifier against ``{base_url}{JWKS_PATH}``.
 58
 59        Args:
 60            base_url: The AXIAM server's base URL; ``JWKS_PATH`` is appended
 61                after stripping any trailing slash.
 62            lifespan: Normal (non-forced) JWKS cache TTL in seconds, passed
 63                through to :class:`~jwt.PyJWKClient`'s ``cache_jwk_set``
 64                lifespan (default :data:`_DEFAULT_LIFESPAN_SECONDS`).
 65        """
 66        jwks_url = base_url.rstrip("/") + JWKS_PATH
 67        # The per-key LRU cache (opt-in via a separate constructor flag,
 68        # intentionally left at its default/disabled state here) has no
 69        # TTL/expiration (Pattern 5 Pitfall); relying solely on the TTL'd
 70        # jwk_set_cache (cache_jwk_set=True) avoids serving a rotated/revoked
 71        # key indefinitely.
 72        self._client = PyJWKClient(jwks_url, cache_jwk_set=True, lifespan=lifespan)
 73        self._last_forced_refetch: float | None = None
 74        self._refetch_lock = threading.Lock()
 75
 76    def verify(self, token: str) -> dict[str, Any]:
 77        """Verify ``token``'s EdDSA signature against the cached JWKS,
 78        returning the decoded claims dict. Does NOT check ``exp`` — that is
 79        the caller's responsibility.
 80
 81        Rejects any token whose header ``alg`` is not ``EdDSA`` BEFORE any
 82        keyset lookup is attempted (algorithm-confusion defense).
 83        """
 84        header = jwt.get_unverified_header(token)
 85        if header.get("alg") != "EdDSA":
 86            raise ValueError(f"unexpected alg {header.get('alg')!r}: only EdDSA is accepted")
 87
 88        signing_key = self._get_signing_key(token)
 89
 90        return jwt.decode(
 91            token,
 92            signing_key.key,
 93            algorithms=["EdDSA"],
 94            options={"require": ["sub"]},
 95        )

 97    def _get_signing_key(self, token: str) -> Any:
 98        """Resolve the signing key for *token*, single-flighted through
 99        ``_refetch_lock`` so a burst of concurrent callers against a cold
100        or stale cache — forced-refetch or not — collapses to exactly one
101        network fetch (D-08/D-09, PERF-03).
102
103        The lock spans the ENTIRE lookup-and-fetch sequence, not merely the
104        "should-I-invalidate" decision (the pre-existing TOCTOU gap this
105        closes): every waiter, once it acquires the lock, first retries
106        against whatever a prior lock-holder may have already
107        fetched/repaired before ever triggering another fetch itself.
108        """
109        with self._refetch_lock:
110            try:
111                return self._client.get_signing_key_from_jwt(token)
112            except PyJWTError:
113                # Unknown kid, a stale in-TTL cache after key rotation, or
114                # an empty/malformed keyset response (e.g. a rotation
115                # window where the new key has not yet propagated).
116                # PyJWTError is the common base of both PyJWKClientError
117                # (unknown kid) and PyJWKSetError (empty/invalid keyset) —
118                # both failure modes warrant the same forced-refetch
119                # response below.
120                pass
121
122            now = time.monotonic()
123            if (
124                self._last_forced_refetch is not None
125                and now - self._last_forced_refetch < _FORCED_REFETCH_MIN_INTERVAL_SECONDS
126            ):
127                # Rate-limited: another caller already forced a refetch
128                # recently (while we waited for the lock, or just before
129                # us) — surface whatever the current cache yields rather
130                # than hammering the endpoint again.
131                return self._client.get_signing_key_from_jwt(token)
132
133            # Force exactly one rate-limited refetch, then retry once. A
134            # second failure propagates to the caller.
135            self._client.jwk_set_cache = None
136            self._last_forced_refetch = now
137            return self._client.get_signing_key_from_jwt(token)

332

336

338

344            ),

349    ``resource_param`` (a path parameter name), or ``resolver`` (a
350    ``request -> str`` callback for anything else — body fields, headers,
351    composite lookups) must be supplied (§11.2.3); this is validated eagerly
352    at factory-construction time, not per-request.
353
354    The returned dependency:
355    1. Authenticates the request via :func:`_authenticate` (401 on failure).
356    2. Resolves and validates the resource UUID (400 on a missing/unparseable
357       value, §11.2.3).

366       obtain a decision at all) -> **fail closed** with ``HTTPException`` 503
367       ``authz_unavailable`` (§11.2.5) — never allow, never retry beyond the
368       client's own bounded single-flight refresh.

370       the caller's identity (roles etc.) without a second lookup.
371

373    ``check_access`` round-trip. Deny/error paths never log or echo the raw
374    token (§11.2.8).

Factory returning a Depends(...)-compatible async dependency that requires an AXIAM authorization check for action on a resource resolved from the request (CONTRACT.md §11).

Composes with, and runs strictly after, the same authentication pipeline require_authenticated_user() uses (_authenticate() — never a separate/duplicated token-verification path, §11.2.1).

Usage::

@app.get("/docs/{doc_id}")
async def get_doc(
    doc_id: str,
    user: AxiamUser = Depends(
        require_access(verifier, "acme", client, "read", resource_param="doc_id")
    ),
):
    ...

Exactly one of resource_id (a literal UUID, for singleton resources), resource_param (a path parameter name), or resolver (a request -> str callback for anything else — body fields, headers, composite lookups) must be supplied (§11.2.3); this is validated eagerly at factory-construction time, not per-request.

The returned dependency:

  1. Authenticates the request via _authenticate() (401 on failure).
  2. Resolves and validates the resource UUID (400 on a missing/unparseable value, §11.2.3).
  3. Calls client.check_access(action, resource_id, scope=scope, subject_id=<authenticated user_id>) — subject_id is always the request's authenticated caller, never this client's own (often service-account) identity (§11.2.2).
  4. Denied (allowed=False or an AuthzError from the server, e.g. the client lacks authz:check_as) -> HTTPException 403 authorization_denied.
  5. A transport failure (AuthError/NetworkError — the SDK could not obtain a decision at all) -> fail closed with HTTPException 503 authz_unavailable (§11.2.5) — never allow, never retry beyond the client's own bounded single-flight refresh.
  6. On success, returns the same AxiamUser so the handler keeps the caller's identity (roles etc.) without a second lookup.

No decision caching (§11.2.6): every call performs a fresh check_access round-trip. Deny/error paths never log or echo the raw token (§11.2.8).
